Week 1 Lecture: Introduction

C# and .NET

  • C# is the language, .NET is the framework
  • C# programs are compiled into .NET binaries and executed through that framework

Interpreted:

  • Java Script, Python etc
  • You write the source code, and that source file will run as is. It is not compiled into an executable.

Compiled to Native Code:

  • You write the source code, which is then compiled. But the compiled code will only run on the platform it is compiled for.
  • E.g. a C# program might be compiled to run on windows 64-bit systems, and will not run on other platforms. This is becuase compiled low level code is native, and

Compiled to Intermediate Bytecode:

  • Bytecode can be efficiently interpreted by a range of systems.
  • It is not a native executeable, it is designed to be interpreted very quickly.
  • It sometimes looks like a native executeable, but it is not.

Dynamic vs Static

  • Dynamic languages are loosely typed, e.g. any python variable can be declared with any data.
  • Static languages are strongly typed, each variable type will only hold certain kinds of data.
  • Using 'var' in C# just infers the type based on the value initialization, it is NOT loosely typed. It doesn't change the fact you need to settle on a type upfront. E.g. if var is initialized with an integer, it can't later be appended with a string.

Notes On C#

  • Strongly typed.
  • Static.
  • Int64 is the largest integer type.
  • 'Method' and 'Function' are synonomous.
  • Argument and return types must be declared with the function/method.
    • Having types declared upfront like this is faster, since the native machine code will be less ambiguous.
